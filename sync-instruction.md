# Синхронизация фронтенда между Replit и основным проектом

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│  REPLIT (редактирование фронта)                                 │
│  GitHub: rakhnianskii9/fb-front-sync                            │
└───────────────────────────┬─────────────────────────────────────┘
                            │ git push/pull
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│  ЭТОТ СЕРВЕР: /home/projects/fb-front-sync (синхронизация)      │
└───────────────────────────┬─────────────────────────────────────┘
                            │ rsync (sync.sh)
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│  ОСНОВНОЙ ПРОЕКТ: /home/projects/new-flowise                    │
│  ├── бэкенд                                                     │
│  └── packages/fb-front/ ← сюда попадает фронт                   │
└─────────────────────────────────────────────────────────────────┘
```

---

## Быстрые команды

| Сценарий | Команда |
|----------|---------|
| Поменял фронт в **Replit** → нужно в основной проект | `./sync.sh from-replit` |
| Нужно отправить текущее состояние в **Replit** | `./sync.sh to-replit` |

---

## Что делают команды

### `./sync.sh from-replit`
1. `git pull` — забирает изменения из GitHub (Replit)
2. `rsync` — копирует в `/home/projects/new-flowise/packages/fb-front/`

### `./sync.sh to-replit`
1. `rsync` — забирает из основного проекта
2. `git commit` + `git push` — отправляет в GitHub (появится в Replit)

---

## Базовые команды (ручной контроль)

```bash
./sync.sh push    # fb-front-sync → основной проект
./sync.sh pull    # основной проект → fb-front-sync
./sync.sh watch   # автосинхронизация при изменениях
```
